
# 분석 기획서

```
본 프로젝트는 Olist E-Commerce 공개 데이터를 활용하여,
커머스 비즈니스의 핵심 KPI를 표준화하고 데이터 마트를 구축한 뒤,
KPI 시계열 분석과 코호트 분석을 중심으로
비즈니스 상태를 지속적으로 진단할 수 있는 분석 환경을 설계하는 것을 목표로 합니다.

분석에 사용되는 데이터는 2016년부터 2018년까지의
주문, 고객, 상품, 결제, 리뷰, 판매자 정보를 포함하고 있으며,
총 9개의 주요 테이블로 구성되어 있습니다.
주문 단위 트랜젝션 데이터와 고객·상품 차원 데이터를 함께 활용하여
시간 흐름에 따른 KPI 변화와 고객의 장기 구매 패턴 분석이 가능합니다.

본 프로젝트는 단발성 EDA나 결과 보고에 그치지 않고,
SQL 기반 데이터 마트와 분석 모듈을 중심으로 한 구조를 설계하여
대시보드 및 향후 확장 가능한 분석 워크플로우를 구축하는 데 중점을 두었습니다.
```

- **사용 데이터**: Brazilian E-Commerce Public Dataset by Olist

- **데이터 출처**: Kaggle(https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

- **데이터 크기**: 9 Table, 52 col

- **데이터 수집 기간**: 약 2년

## 1. 문제 정의


```
Olist는 매출, 주문 수, 구매자 수와 같은 핵심 KPI를 지속적으로 모니터링하고 
이를 기반으로 의사결정을 수행하고자 한다.
그러나 실제 분석 환경에서 지표가 여러 쿼리와 테이블에 분산되어 있어, 동일한 KPI라도 계산 기준이 일관되지 않게 사용되는 경우가 많다.

이로 인해 단순한 현황 파악조차 매번 새로운 분석 작업이 필요하며,
이전 분석 결과를 재사용하거나 기간별·채널별 비교를 수행하는 데 불필요한 비용이 발생하고 있다.
특히 재구매율, 코호트와 같은 구조적 분석이나 시간 흐름을 고려해야 하는 시계열 분석은
일회성 EDA 방식으로는 지속적인 관리가 어렵다.

따라서 단순 지표 산출을 넘어, 일관된 기준의 KPI를 중심으로 데이터를 구조화하고,
반복적으로 활용 가능한 분석 환경을 구축할 필요가 있다.
```

- **지속적인 모니터링의 어려움**
	- 핵심 지표들이 여러 테이블과 쿼리에 분산되어 있어 지속적으로 활용 가능한 KPI 계산 기준이 일관되지 않음
	
- **불필요한 비용 발생**
	- 단순한 현황 파악조차 매번 새로운 분석 작업이 필요함
	
- **분석을 지속적으로 관리하기 어려움**

## 2. 분석 목적


```
본 프로젝트의 주요 목적은 커머스 비즈니스의 핵심 KPI를 일관된 기준으로 관리하고
이를 기반으로 비즈니스 상태를 지속적으로 진단할 수 있는 분석 환경을 구축하는 것입니다.

이를 위해 다음과 같은 목표를 설정합니다.

- 커머스 비즈니스에서 반복적으로 활용되는 핵심 KPI를 정의하고, 
  계산 기준을 명확히 하여 지표의 일관성을 확보

- 원천 데이터를 분석 목적에 맞게 구조화하여, SQL 기반 데이터 마트를 설계

- KPI 시계열 분석과 코호트 분석을 중심으로, 
  단발성 분석이 아닌 반복적으로 활용 가능한 분석 구조를 구축

- 데이터 마트를 기반으로 대시보드를 구성하여,
  주요 지표를 직관적으로 확인하고 의사결정에 활용할 수 있도록 함
```

- **분석 목적**
	- 커머스 비즈니스의 핵심 KPI를 일관된 기준으로 관리
	- 이를 기반으로 비즈니스 상태를 지속적으로 모니터링할 수 있는 분석 환경 구축

- **분석 목표**
	- 핵심 KPI 정의 및 지표의 일관성 확보
	- SQL 기반 데이터 마트 설계
	- 분석 모듈화를 통한 반복적 활용 구조 구축
	- 대시보드 구성을 통한 주요 지표의 직관적인 확인

## 3. 기대 효과


```
본 프로젝트를 통해 커머스 비즈니스의 핵심 지표를 일관된 기준으로 관리할 수 있는 분석 환경을 구축함으로써,
다음과 같은 효과를 기대할 수 있습니다.

- 핵심 KPI가 표준화된 구조로 관리됨에 따라,
  지표 정의의 혼선 없이 매출, 주문, 구매자 수 등의 현황을 일관된 기준으로 확인할 수 있음

- SQL 기반 데이터 마트와 분석 모듈을 통해,
  반복적인 분석 작업에 소요되는 시간을 줄이고 분석 결과의 재사용성과 유지보수성을 향상시킬 수 있음

- 시계열, 코호트 분석을 통해
  단순 지표 나열이 아닌 비즈니스 흐름 관점에서의 문제 구간과 개선 포인트를 구조적으로 파악할 수 있음

- 대시보드를 활용하여 주요 지표를 직관적으로 모니터링함으로써,
  데이터 기반 의사결정을 보다 빠르고 안정적으로 지원할 수 있음

궁극적으로 본 프로젝트는 단발성 분석 중심의 접근에서 벗어나,
지속적으로 활용 가능한 분석 체계를 구축함으로써
실무 환경에 적용 가능한 데이터 분석 워크플로우를 제시하는 것을 목표로 합니다.
```

- **기대 효과**
	- 지표 정의의 혼선 없이 매출, 주문, 구매자 수 등의 현황을 일관적인 기준으로 확인할 수 있음
	- 반복적인 분석 작업에 소요되는 시간을 줄임
	- 분석 결과의 재사용성과 유지보수성을 향상시킴
	- 비즈니스 흐름 관점에서의 문제 구간과 개선 포인트를 구조적으로 파악할 수 있음
	- 데이터 기반 의사결정을 보다 빠르고 안정적으로 지원할 수 있음

## 4. 분석 범위 및 전제


```
본 프로젝트는 Olist 이커머스 공개 데이터를 기반으로,
커머스 비즈니스의 전반적인 현황을 진단하기 위한 분석 환경을 구축하는 데 초점을 둡니다.
이에 따라 분석 범위와 전제 조건을 다음과 같이 설정합니다.

분석 범위

- 분석 대상 기간은 약 2016년부터 2018년까지의 거래 데이터로 한정합니다.

- 주문, 주문 상품, 고객, 상품, 판매자, 결제, 리뷰 데이터를 주요 분석 대상으로 사용합니다.

- 분석 단위는 주문 및 고객을 기본으로 하며,
  상품 판매자 정보는 보조적인 차원으로 활용합니다.

- 핵심 KPI 시계열 분석과 코호트 분석을 본 프로젝트의 주요 분석 범위로 설정합니다.

분석 전제 및 제외 사항

- 본 프로젝트는 공개 데이터의 특성을 고려하여
  사용자 행동 로그(방문, 클릭, 장바구니 등)을 통해 가능한 퍼널 분석이나 AARRR 프레임워크는 
  메인 분석 범위에서 제외합니다.

- 예측 모델링이나 머신러닝 기반 분석은 본 프로젝트의 범위에 포함하지 않으며,
  데이터 구조 설계와 지표 정의, 분석 환경 구축에 중점을 둡니다.

- 데이터에 포함된 결측치, 이상치, 상태 값의 불완전성은
  분석 목적에 맞게 합리적인 기준을 설정하여 처리합니다.

- 공개 데이터이므로 일부 비즈니스 맥락은 제한적으로 해석할 수 있음을 전제로 합니다.
```

- **분석 범위**
	- 분석 대상 기간: 약 2016년 ~ 2018년
	- 주요 분석 대상: 주문, 주문 상품, 고객, 상품, 판매자, 결제, 리뷰 데이터 등
	- 분석 단위: 주문 및 고객 (보조적으로 상품 판매자 정보 활용)
	- 주요 분석 범위: 핵심 KPI 시계열 분석, 코호트 분석

- **전제 및 제외 사항**
	- 이벤트성 데이터(마케팅 비용, 광고 캠페인, 실시간 이벤트 로그 등)는 분석 범위에서 제외
	- 퍼널 분석, AARRR 분석은 분석 범위에서 제외
	- 예측 모델링이나 머신러닝 기반 분석을 제외하고 데이터 구조 설계와 지표 정의, 분석 환경 구축에 초점
	- 결측치, 이상치, 불완전성 등은 분석 목적에 맞게 합리적인 기준을 설정하여 처리
	- 일부 비즈니스 맥락은 제한적으로 해석

## 6. 분석 Flow


```
본 프로젝트는 원천 데이터부터 대시보드까지의 분석 과정을 단계별로 분리하여 설계함으로써, 
분석 로직의 명확성과 재사용성을 확보하는 것을 목표로 합니다.

전체 분석 Flow는 다음과 같은 구조로 구성됩니다.

Raw Data -> Staging -> Data Mart -> Analysis Modules -> Dashboard
```

- **Raw Data**
	- Olist에서 제공하는 원본 CSV 데이터를 그대로 적재
	- 데이터의 원형을 보존하며 직접적인 수정은 수행하지 않음

- **Staging**
	- Raw 데이터의 구조와 내용을 기반으로, 분석에 필요한 최소한의 정제 및 표준화를 수행
	- 데이터 타입 변환, 컬럼명 정리, 상태 값 표준화, 기본적인 결측치 및 이상치 처리 등으로 한정

- **Data Mart**
	- 분석 목적에 맞게 데이터를 구조화한 핵심 계층으로, Fact 테이블과 Dimension 테이블 그리고 집계 마트로 구성
	- 핵심 KPI 계산 기준을 반영하여 매출, 주문 수, 구매자 수 등의 지표를 일관된 방식으로 산출
	- 데이터 마트는 이후 분석 모듈과 대시보드의 단일 데이터 소스로 활용

- **Analysis Modules**
	- 코호트 분석을 메인 분석으로 KPI 시계열 분석까지 진행
	- 추가로 주문 취소율을 중심으로 한 거래 안정성 점검을 보조 분석으로 사용
	- 각 분석은 재사용 가능한 형태로 설계하여, 기간별·조건별 비교 분석이 가능하도록 설계

- **Dashboard**
	- 분석 모듈과 데이터 마트를 기반으로 주요 KPI와 분석 결과를 시각화하여 제공
	- 대시보드를 통해 비즈니스 현황을 직관적으로 확인하고, 문제 구간을 빠르게 파악
